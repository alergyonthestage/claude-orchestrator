# Config Editor Safety Rules

## Before Modifying Files
- Check if the file exists before writing — show diff if overwriting
- For project.yml edits, validate YAML structure after changes
- For pack.yml edits, remind user to run `cco pack validate` on host

## Protected Paths
- NEVER modify `.cco/` directories (framework-managed metadata)
- NEVER modify `secrets.env` without explicit user request
- NEVER delete projects or packs without explicit confirmation
- NEVER modify `manifest.yml` directly — it's auto-generated by cco

## Vault Awareness
- After significant changes, remind user: `cco vault sync` on host
- Before destructive changes, suggest: `cco vault diff` to review state
- If user mentions sharing/publishing, reference the vault and Config Repo docs

## cco Commands
- cco CLI commands CANNOT run inside this container — they are host-only
- When an action requires cco, show the exact command for the user's
  host terminal and explain what it does
