# Documentation-First Rule

Before starting analysis, design, or implementation on any topic:

1. Check if relevant documentation already exists:
   - Design documents and ADRs in `docs/`
   - Prior analysis or decisions on the same topic
   - Implementation status and roadmap entries
   - Related rules in `.claude/rules/`

2. Read existing documents before proposing solutions — they contain
   decisions, constraints, and rationale that must inform your work.

3. Reference existing documents in your analysis/design. Build on prior
   work rather than starting from scratch.

If documentation exists but is outdated, note the discrepancies and
propose updates — do not silently ignore prior decisions.
