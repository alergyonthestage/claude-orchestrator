# claude-orchestrator Documentation

The documentation is split by audience. Pick the entry point that matches what
you want to do.

| Audience | Start here | What you'll find |
|---|---|---|
| **Users** — you want to run cco and configure projects | [users/README.md](users/README.md) | A learning path: install, first project, key concepts, then per-domain guides and references (configuration, packs, integrations, environment, security, CLI). |
| **Maintainers** — you develop or contribute to cco | [maintainers/README.md](maintainers/README.md) | The domain index: architecture, ADRs, design docs, the roadmap, and engineering conventions. |

## Layout

- `users/` — task-oriented guides and references for people using cco.
- `maintainers/` — design docs, analyses, ADRs, reviews, and the roadmap for
  people building cco.
- `archive/` — superseded design trees kept for reference (e.g. the former
  vault, sharing, and resource-lifecycle designs). These do **not** describe the
  current system; consult them only to understand prior decisions. See
  [archive/](archive/).

> New to cco? The fastest start is the repo [README](../README.md) quickstart,
> ending with `cco start tutorial`.
